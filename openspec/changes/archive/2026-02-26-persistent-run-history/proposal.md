## Why

Simulation run data is lost on restart because the default store is in-memory. The Neo4j store exists for `SimulationRunRecord` but benchmark and experiment reports are still held in `ConcurrentHashMap` even when Neo4j mode is active. This makes it impossible to compare runs over time, track regression trends, or share results across sessions.

## What Changes

- **Switch default store to NEO4J** — `dice-anchors.run-history.store` defaults to `NEO4J` instead of `MEMORY`
- **Persist benchmark reports to Neo4j** — `Neo4jRunHistoryStore` stores `BenchmarkReport` as graph nodes (currently in-memory `ConcurrentHashMap`)
- **Persist experiment reports to Neo4j** — same treatment for `ExperimentReport`
- **Persist baselines to Neo4j** — scenario baseline references stored as graph relationships instead of in-memory map
- **Safe run naming for parallel execution** — run IDs generated by `BenchmarkRunner` MUST be unique even under concurrent `StructuredTaskScope` execution (current 8-char UUID substring is already safe, but benchmark report IDs need validation)
- **Run History panel in SimulationView** — surface historical runs directly in the sim UI, not just via dialog; allow filtering by scenario, date range, and injection status
- **Cross-session comparison** — RunInspectorView comparison mode works with persisted historical runs, not just runs from the current session

## Capabilities

### New Capabilities

- `run-history-ui`: Dedicated run history browsing panel with filtering, sorting, and comparison launch — replaces the current modal dialog approach

### Modified Capabilities

- `run-history-persistence`: Default store changes from MEMORY to NEO4J; benchmark/experiment reports and baselines gain full Neo4j persistence
- `run-history`: Interface contract unchanged, but the Neo4j implementation now covers all report types (not just `SimulationRunRecord`)

## Impact

- **`Neo4jRunHistoryStore`** — add Cypher queries for `BenchmarkReport`, `ExperimentReport`, and `Baseline` node/relationship persistence
- **`SimulationConfiguration`** — flip default from `MEMORY` to `NEO4J`
- **`SimulationRunStore` (memory impl)** — unchanged, still available for lightweight/test use
- **`RunHistoryDialog` / `SimulationView`** — replace or augment modal with inline history panel
- **`RunInspectorView`** — ensure comparison loads from persistent store (already does via `RunHistoryStore`, but verify no session-local assumptions)
- **`application.yml`** — update default config value
- **Neo4j schema** — new node labels `BenchmarkReport`, `ExperimentReport`; new relationship type for baseline references
